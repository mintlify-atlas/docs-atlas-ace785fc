**coss ui** is the evolution of **Origin UI**, an open-source collection of React and Tailwind components created by Pasquale Vitiello and Davide Pacilio. Origin UI was recently **acquired by Cal.com** and grown into a broader, more ambitious ecosystem, with Pasquale contributing directly to the development of the new library.

The goal is to bring together the best parts of Origin UI with a stronger foundation — Base UI primitives — so developers get not just components, but a layered system of **primitives**, **particles**, and **atoms**.

## Origin UI legacy

Origin UI offered a large collection of 600+ ready-to-use components built on Radix UI. With the transition to coss ui:

- Existing Origin UI components are evolving into **particles** — higher-level UI patterns composed of Base UI primitives.
- The original Origin UI collection remains available on the [coss.com origin page](https://coss.com/origin), so developers can continue to access it during the transition.

<Note>
  Origin UI components are still available and functional. Active development now focuses on the new coss ui primitives and particles.
</Note>

## What's next

- **Expand primitives** — ensure full coverage of common UI needs with more Base UI-powered components.
- **Add particles** — grow the particles collection to match and eventually surpass the original Origin UI catalog.
- **Introduce atoms** — API-enhanced components that bring external integrations (scheduling, auth, notifications) out of the box.
