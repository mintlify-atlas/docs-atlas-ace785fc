## Overview

`Field` wraps Base UI's `Field` primitive family. It connects a label, an input, a description, and an error message using accessible HTML attributes automatically. The `FieldError` component only renders when the associated control is invalid.

## Installation

```bash
npx shadcn@latest add @coss/field
```

## Basic usage

```tsx
import {
  Field,
  FieldDescription,
  FieldError,
  FieldLabel,
} from "@/components/ui/field"
import { Input } from "@/components/ui/input"
```

```tsx
<Field>
  <FieldLabel>Email</FieldLabel>
  <Input name="email" type="email" required />
  <FieldDescription>We’ll never share your email.</FieldDescription>
  <FieldError>Please enter a valid email address.</FieldError>
</Field>
```

## Props

### Field

Root component. Renders a `<div>` with flex column layout.

<ParamField path="name" type="string">
  Associates this field with a form control by name. Used by `Form` to surface server errors.
</ParamField>

<ParamField path="...props" type="FieldPrimitive.Root.Props">
  All props from Base UI's `Field.Root` are supported.
</ParamField>

### FieldLabel

Label for the field control. Styled wrapper for `Field.Label` from Base UI.

<ParamField path="...props" type="FieldPrimitive.Label.Props">
  All props from Base UI's `Field.Label` are supported.
</ParamField>

### FieldDescription

Supplementary description text rendered below the control.

<ParamField path="...props" type="FieldPrimitive.Description.Props">
  All props from Base UI's `Field.Description` are supported.
</ParamField>

### FieldError

Error message rendered when the field is invalid. Hidden when the field is valid.

<ParamField path="show" type='"valid" | "invalid" | "always"'>
  Controls when the error is shown. Defaults to `"invalid"`.
</ParamField>

<ParamField path="...props" type="FieldPrimitive.Error.Props">
  All props from Base UI's `Field.Error` are supported.
</ParamField>

### FieldItem

A flex container for inline field layouts.

### FieldControl

Alias for `Field.Control` from Base UI. Forwards `aria-invalid`, `aria-describedby`, and `id` to a custom control.

### FieldValidity

Alias for `Field.Validity` from Base UI. Renders children with the current validity state.
