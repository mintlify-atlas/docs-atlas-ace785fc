**coss ui** is a collection of beautifully designed, accessible, and composable components for your React apps. Built on top of [Base UI](https://base-ui.com/) and styled with [Tailwind CSS](https://tailwindcss.com/), it's designed for you to copy, paste, and own.

We think Base UI is the best foundation for modern web applications. We've taken its powerful, unstyled primitives and given them a design system that's ready to go, right out of the box.

This is the component library being progressively adopted at [Cal.com](https://cal.com). We're building it in the open for anyone who wants to create beautiful, reliable user interfaces.

<Warning>
  coss ui is currently in early development. We're building this library in the open and actively working on new components and features. Base UI itself is also in beta, so you may encounter breaking changes as both projects evolve. We recommend using this in projects where you're comfortable adapting to changes.
</Warning>

## How it works

Our approach is simple: you should own your code. Inspired by the copy-paste ethos of **shadcn/ui**, you get the source — no black-box abstractions.

## Primitives, particles, and atoms

At the core of coss ui are **UI primitives** — unstyled, accessible building blocks powered by Base UI. These handle accessibility, focus management, and keyboard interactions without dictating design.

On top of primitives, we introduce [particles](/particles/overview). Particles are **pre-assembled components** — authentication forms, tables, date pickers — that combine multiple primitives into ready-to-use solutions. They're opinionated in design, but easy to customize, extend, or break apart.

Finally, [atoms](https://cal.com/atoms) are **API-enhanced particles**: components that connect UI to external data and services. Think smart components with out-of-the-box behaviors for common patterns like authentication flows and scheduling.

Together, this gives you a full spectrum of abstraction levels:

| Layer | Role |
|---|---|
| **Primitives** | The foundation — accessible, unstyled Base UI building blocks |
| **Particles** | The patterns — pre-assembled UI ready to drop in and customize |
| **Atoms** | The integrations — particles connected to APIs and external services |

Choose the right level for your project, whether you need a bare primitive, a polished UI pattern, or a complete API-ready solution.

## Built on Base UI, designed for production

coss ui is built on **Base UI** from the ground up — not as an adaptation from Radix UI. We chose Base UI for its robust, accessible, and unopinionated primitives that handle the hard parts without imposing design decisions.

Our components go beyond base primitives, incorporating:

- **Production-tested patterns** from real-world applications at Cal.com and coss.com
- **Extended component variants** for common use cases
- **Thoughtful defaults** optimized for modern, dense UIs
- **Consistent design system** across all components

<Info>
  If you're migrating from shadcn/ui or Radix UI, see the [migration guide](/guides/radix-migration) for a smooth transition.

## Built for humans and AI

Components are written to be clear, readable, and predictable, so language models can understand, reason about, and modify them. Every page includes a **Copy Markdown** button and we publish a `llms.txt` map of the docs for your AI tools.

## Open source, made sustainable

This project is open source and part of the **COSS** (Commercial Open Source Software) philosophy. We build in public, maintain transparency, and believe open source works best when it's sustainable.

As part of [coss.com's mission](https://coss.com), we support open source projects so they can grow, maintain long-term viability, and provide value for users and contributors alike.
